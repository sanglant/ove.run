# ove.run

## Your control plane for AI coding agents.

Run Claude, Gemini, Copilot, and Codex side by side. Orchestrate multi-step workflows. Stay in control without babysitting.

ove.run is a terminal-first desktop app that turns AI coding agents into a managed, observable development workflow. Delegate ambitious tasks, watch agents work in real time, and build institutional memory that compounds over time.

---

## Features

### Loop Engine
#### Autonomous multi-step orchestration

Describe what you want to build. The Loop Engine decomposes your request into prioritized stories with acceptance criteria, assigns them to agents, runs quality gates after each iteration, and decides whether to proceed, retry, or pause.

No manual prompt engineering per task. No copy-pasting between tools. One request in, working code out.

**How it works:**
- **Plan** — Your request is broken into atomic, testable stories with priorities and dependencies
- **Execute** — An agent session is spawned for each story with full context injection
- **Verify** — Configurable quality gates run automatically: build, lint, typecheck, tests
- **Judge** — The Arbiter reviews results against acceptance criteria and decides next steps
- **Iterate** — Failed stories retry with modified context. A circuit breaker prevents infinite loops

**What makes this different:** Most AI tools run one prompt at a time. The Loop Engine runs a full development cycle — plan, execute, verify, judge — autonomously. It handles story dependencies, tracks iteration attempts, and knows when to stop.

### Multi-Agent Terminal
#### Run any agent. Run them all at once.

First-class support for Claude Code, Gemini CLI, GitHub Copilot, and Codex CLI — all in one interface. Split your screen, drag tabs between panes, and watch multiple agents work in parallel.

- **Split-screen layouts** with drag-to-resize panes
- **Drag-and-drop tab reordering** across panes
- **Live status detection** — idle, working, needs input, finished, error
- **YOLO mode toggle** per session for autonomous execution
- **Session persistence** — resume where you left off after restart
- **Per-agent configuration** — custom CLI args, environment variables, model selection

**What makes this different:** Other tools lock you into one provider. ove.run treats agents as interchangeable. Run Claude on your backend refactor and Gemini on your frontend migration at the same time. Compare approaches. Use the right agent for each job.

### Arbiter System
#### Configurable autonomy with three trust levels

The Arbiter is the decision layer between you and your agents. It monitors sessions for questions and permission prompts, reviews context, and responds based on a trust level you control.

| Level | Name | Behavior |
|---|---|---|
| 1 | Supervised | Approves each step with you |
| 2 | Autonomous | Runs freely, asks when stuck |
| 3 | Full Auto | Handles everything |

When an agent needs input, a feedback modal surfaces the terminal output, parses available options into clickable buttons, and starts a configurable countdown. If you don't respond, the Arbiter answers based on the trust level. If you do, it steps aside.

Every Arbiter decision is logged with full reasoning. You can audit exactly why it approved, rejected, or retried any story.

**What makes this different:** Most AI tools are either fully autonomous (scary) or fully manual (tedious). The Arbiter gives you a dial. Turn it up as trust grows. Turn it down for sensitive work. The reasoning log means you always know why.

### Context Store
#### Progressive detail with L0 / L1 / L2 layers

Context units are reusable blocks of knowledge that shape how agents work. They come in four types: Personas, Skills, Knowledge, and References. Each unit stores content at three levels of detail:

- **L0 — Summary**: One sentence. Used for quick relevance checks.
- **L1 — Overview**: Structured paragraph. Key points and approach.
- **L2 — Content**: Full markdown. The complete specification.

Write the L2 content. L0 and L1 are auto-generated by the Arbiter from your content — one click, no manual summarization.

**12 built-in personas** ship out of the box: Backend Developer, Frontend Developer, Security Auditor, Code Reviewer, DevOps Engineer, Database Architect, and more. **8 built-in skills** cover Code Review, Testing, Git Workflow, Error Handling, Performance, Security, API Design, and Documentation.

Context units can be scoped globally or per-project, assigned as project defaults, or attached to individual sessions.

**What makes this different:** Other tools give you a system prompt box. ove.run gives you a structured context library with progressive detail, auto-summarization, and per-session assignment. Agents get exactly the context they need — no more, no less.

### Agent Memory
#### Persistent learning that compounds over time

After each agent session, the system extracts facts, observations, and learnings from the terminal output. These raw memories are tagged with entities, topics, and an importance score (0–1 scale).

When enough memories accumulate (10+), a consolidation pass synthesizes them into higher-level insights. Consolidated memories are injected as L0 summaries into future story prompts, so agents build on previous work instead of starting from scratch.

- **Automatic extraction** from agent output
- **Entity and topic tagging** for searchability
- **Importance scoring** with decay over time
- **Consolidation** into synthesized insights
- **Visibility control** — private or public per memory
- **Full-text search** across all memories

**What makes this different:** Every other tool treats each session as isolated. ove.run builds institutional memory. The more you use it, the smarter your agents become. Context from last week's refactor informs this week's feature build.

### Bug Tracking
#### Connect your issue tracker. Delegate to agents.

OAuth-based integration with Jira, GitHub Issues, and YouTrack. Browse issues, view details, and spawn agent sessions to work on specific bugs — one click from issue to terminal.

- Zero-friction OAuth setup
- Filterable issue lists with priority and status
- Full issue detail view

**What makes this different:** No API keys. No config files. OAuth in, browse issues, click "delegate." The agent gets the bug description as its prompt. That's it.

A full markdown editor with rich text formatting, split view, and raw source mode. Notes can be toggled into agent context, so your project briefs, architecture decisions, and meeting notes directly inform how agents work.

### Quality Gates
#### Automated verification before arbiter judgment

Configure build, lint, typecheck, and test commands per project. The Loop Engine runs them automatically after each story iteration. Only stories that pass all gates reach the Arbiter for final judgment.

No broken code slips through. The gates catch it before the Arbiter even looks.

---

### Stats Dashboard
#### Per-project usage analytics

Track sessions by agent type, loop iterations, story completion rates, context unit usage, memory growth, and consolidation activity. All scoped per project.

Skills show L0/L1/L2 completeness levels. Stories show status breakdown. The Arbiter section shows trust level and activity state.

When the Loop Engine is running, a floating console widget appears in the corner — visible from any panel. See live agent output, the current activity message, and expand to a full modal view. Click through to the Loop panel when you need the full picture.

### Notifications

Desktop and in-app notifications when agents finish work or need input. Unread count badge, notification history, and one-click jump to the relevant session.

Interactive walkthroughs for the terminal, git, context, bugs, notes, and loop panels. First-time users get oriented without reading docs.

## Design

Dark mode only. Amber accent. Terminal-inspired, not SaaS-generic.

ove.run follows five design principles:

1. **Control over spectacle** — Visual choices reinforce trust, legibility, and operator confidence
2. **Technical, not generic** — Developer-native aesthetic with terminal cues and system-like structure
3. **Power made approachable** — Complex capabilities feel understandable at a glance
4. **Dark with disciplined warmth** — Dark foundation with consistent amber accent signature
5. **Motion with respect** — Animation clarifies and elevates, never distracts

---

## Technical Foundation

- **Rust backend** — Native performance, low resource usage, local execution
- **React + TypeScript frontend** — Modern component architecture
- **SQLite + FTS5** — Local database with full-text search for memories and context
- **PTY management** — Direct terminal control via pseudo-terminal multiplexing
- **Event-driven architecture** — Real-time updates between backend and UI
